# jsc_google_idp (Resource)

Manages a Google Workspace identity provider connection in JSC. Used to sync users and groups from Google Workspace for risk-based policies and ZTNA app access.

## OAuth Consent Requirement

Google Workspace IdP connections require a manual consent step that cannot be automated. During `terraform apply`, the resource:

1. Creates the Google Workspace connection in JSC
2. Triggers a consent transaction to generate a Google OAuth URL
3. Stores the URL in the `consent_url` attribute — visible in apply output and retrievable with `terraform output google_consent_url`
4. Visit the URL in a browser to grant consent (requires Google Workspace admin privileges)
5. After completing consent, run `terraform refresh` — `state` updates to `APPROVED` and `consent_url` is cleared from state automatically

### Required Google Workspace Scopes

The consent flow requests these OAuth scopes:
- `admin.directory.group.readonly` — Read group membership
- `admin.directory.user.readonly` — Read user directory

## Notes

- Update is implemented as delete + recreate — a new consent URL will be generated
- If the connection is deleted outside Terraform, the next `terraform plan` will detect drift and offer to recreate it
- Connection states: `INITIAL` → `PENDING` → `APPROVING` → `APPROVED` | `DENIED`
- Supports `terraform import` for existing connections

## Example Usage

```hcl
resource "jsc_google_idp" "google_connection" {
  name = "Google Workspace - Production"
}

output "google_idp_state" {
  value = jsc_google_idp.google_connection.state
}

output "google_consent_url" {
  value     = jsc_google_idp.google_connection.consent_url
  sensitive = true
}
```

## Import

Existing Google Workspace IdP connections can be imported using the connection ID:

```shell
terraform import jsc_google_idp.google_connection <connection-id>
```

The connection ID can be found in the RADAR console URL when viewing the connection.
